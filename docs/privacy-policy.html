Veil — Privacy Policy

Last updated: June 2026

Veil is a browser extension that enforces ad frequency caps using local differential privacy. This policy explains exactly what data is processed, what is stored, and what is never collected.

What Veil collects

Data Where it goes Retention
cohort_id
e.g. us-unknown-desktop
Derived from timezone region + screen width. Represents a large group, not an individual.		 Stored in browser memory. Sent to cap-service only if server mode is enabled. In-memory only. Cleared on service worker restart.
campaign_id
e.g. google_ads_iframe_/7646/...
Ad slot identifier extracted from ad element attributes. Identifies the ad, not the user.		 Stored in browser memory. Sent to cap-service only if server mode is enabled. In-memory only. Cleared on service worker restart.
noisy_value
e.g. 3
A mathematically scrambled impression count. The true count is never transmitted — noise is added before any data leaves the device.		 Sent to cap-service only if server mode is enabled. Stored in Postgres impression_log. 24-hour window, then expires.
Session stats
totalServed, totalSuppressed, recentActions
Aggregate counts used to display the popup badge and recent activity.		 Stored locally in chrome.storage.local. Never transmitted. Until user clears extension data.

What Veil never collects

DataStatus
User identifier of any kindNever collected. No user_id column exists in any table. Structurally impossible.
True impression countNever transmitted. Noise is applied before any network call. The server receives only the scrambled value.
Browsing historyNever collected. The content script checks for ad slot presence only.
Page content or URLsNever read or transmitted.
IP addressNot stored by the cap-service. Standard server logs are not retained.
Cross-site identityCohort is derived from timezone + screen width only. Cannot be used to track individuals across sites.

Local-only mode (default)

By default, Veil runs entirely on your device. No data is sent to any server. Frequency cap decisions are made locally using an in-memory count. Nothing leaves your browser.

Server mode is opt-in and requires you to manually enter a server URL in the extension settings. Without a configured server URL, no network requests are made.

Third-party services

Veil does not use analytics, crash reporting, or any third-party SDKs. No data is shared with ad networks, data brokers, or any third party.

Children

Veil does not knowingly collect data from children under 13.
